It's about interacting via the web to a simple database of 3 tables: Customer, Product, Purchase. The app was written with the idea of maintaining the database as an Officer or an Admin. An OFFICER is able to perform Create, Read, Update operations on tables Customer and Product, and to read Purchase. An ADMIN is able to perform Create, Read, Update and Delete operations to all tables. There are two default accounts, one-per-role, with user-password pairs "officer", "officer" and "admin", "admin". Unauthenticated users can only read Product table. At the moment it is not possible to register new accounts.
